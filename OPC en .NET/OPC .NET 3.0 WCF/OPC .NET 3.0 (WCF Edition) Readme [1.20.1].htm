OPC .NET 3.00 (WCF Edition)

Build 120

November 12, 2010

 

Overview

 

This package contains the source files for the "OPC .NET 3.0 (WCF Edition)" (formally known as OPC Xi). These examples can be used to create WCF applications which can wrap existing OPC Classic products.

 

The term Xi is retained the namespace prefix in the code base.

 

Change Notes

 

This update include documentation changes and minor big fixes.

 

Distribution

 

The "OPC .NET 3.0 (WCF Edition)" is distributed as three packages of source code:

 

1) Contracts and ServerBase

2) Client Source Code

3) All Source Code

 

Each package is a ZIP archive containing the source code tree.

 

The source code is classed as “sample code” and will need to be adapted to meet the needs of different vendors.

Contents

 

This folder contains the source code for all OPC Xi Projects. The Xi TextFiles folder contains example text files that are used by the client programs to configure Xi Data Lists. Xi Data Lists correspond to OPC DA Groups. All other folders contain Xi Visual Studio 2008 projects.

 

The projects are organized into the following types:

A brief description of each follows:

 

Xi Client Base

Implements a common base of client code that interacts with WCF and implements the Xi Client Base API.  It also contains a simplified API that is contained in Xi Client Base/API Classes. 

 

The API Classes are built on top of the Xi Client Base API, and provide an interface to client applications composed of a list of Xi Servers that is populated by the Client Base, and that allows the client application to create Subscriptions for these Xi Servers.

 

Each Subscription represents a list of data items, a list of events/alarms, or a list of historical data items that the client application can read, write, or to which the client application can subscribe.  If the client application subscribes to a subscription, the client base uses callbacks to the client application to deliver changed values and alarms/events.  These values/alarms/events are received from the Xi server in callbacks when the connection to the server uses namedPipes or TCP, and in Poll responses when HTTP is used.

 

Xi Client Base API

Defines a standard API for client applications to use.

 

Xi Client Credentials

This is a separate project that allows the client application vendor to customize how the client credentials are obtained when Windows credentials cannot be used. The Client Base calls this class library (dll) to obtain the client credentials. The current implementation displays a dialog box to prompt the user for user name and password, but custom implementations might be written to use a different prompting mechanism or to read the credentials, for example, from a smart card.

 

Xi Client Dialogs

This project defines a number of common dialog boxes that can be used by client applications. They are used by the client exe programs contained in this Solution.

 

Xi Client GenericApp

This Windows Forms application exercises the ClientBase API. It is designed for programmers to show how to use this API. Each screen operation (e.g. button-click) maps closely to an API call.

 

Xi Client GenericApp AdhocApi

 

This Windows Forms application exercises the ClientBase API Classes interface defined in the XiAdhocDataSubscription class. The XiAdhocDataSubscription class is based on the XiDaSubscription class defined within the ClientBase API Classes.

 

This XiAdhocDataSubscription class is designed for client applications that wish to read and/or write (but not subscribe) using only the Id of the item.

 

The XiAdhocDataSubscription class automatically manages the Xi Lists for the client application, thus relieving the client application from having to create Xi Lists and add items to them. If an item is not read or written within a timeout period specified by the client application, the item is automatically removed from the Xi List maintained by XiAdhocDataSubscription

class. 

 

Xi Client GenericApp ClassApi

This Windows Forms application exercises the ClientBase API Classes interface that defines a ServerList object and a SubscriptionList object. This interface is designed to simplify client access to the Xi Server.

 

Xi Common Support

This class library contains classes/methods that are used by both the client and server.

 

Xi Contracts

This project defines the formal WCF interface for Xi.

 

Xi Contracts Constants

Enumerations are passed either as uints or strings in the Xi Contracts interfaces. This project defines standard values for these enumerations. It is separate from the Xi Contracts so that it can be updated without making changes to the Xi Contracts.

 

Xi OPC COM API

This project defines the interface between the Xi Wrapper code and the OPC COM calls. It decouples the Xi Wrapper code from the implementation of the C++ calls to the OPC COM servers.

 

Xi OPC COM Impl

This project is a C++/CLI project that bridges between managed and unmanaged code. It implement the Xi OPC COM API that can be called from managed code, and makes calls to the OPC COM servers. 

 

Xi OPC Wrapper Impl

This project contains the logic that maps the OPC Xi interfaces onto the OPC COM interfaces. It is written as a subclass of the Xi Server Base classes.

 

Xi Server Base

This project provides standard handlers for the Xi WCF interfaces, and defines abstract methods that it calls for implementation specific handling of the Xi interface methods. It has been designed to support both OPC COM wrappers and native Xi servers. As such, it codifies the expected behavior of the

Xi server interfaces.

 

Xi ServiceHost DiscoveryServer ConsoleApp

This server application is a console application implementation of the Xi Discovery Server. This server can operate using PNRP and/or a manual configuration file to discover Xi servers. It implements the Xi IServerDiscovery interface that allows clients to retrieve a list of Xi servers to which they can connect.

 

Xi ServiceHost DiscoveryServer WinService

This server application is a Windows Service implementation of the Xi Discovery Server. This server can operate using PNRP and/or a manual configuration file to discover Xi servers. It implements the Xi IServerDiscovery interface that allows clients to retrieve a list of Xi servers to which they can connect.

 

Xi ServiceHost OPCWrapper ConsoleApp

This server application is a console application implementation of the Xi server that wraps OPC COM wrapper servers. Each instance of this server can wrap one OPC DA, HDA, and/or A&E server. It app.config file is used to specify the prog ids

of each wrapped server, and optionally the machine name of each wrapped server if the wrapped server is not on the same machine as the Xi wrapper.

 

Xi ServiceHost OPCWrapper WinService

This server application is a Windows Service implementation of the Xi server that wraps OPC COM wrapper servers. Each instance of this server can wrap one OPC DA, HDA, and/or A&E server. It app.config file is used to specify the prog ids of each wrapped server, and optionally the machine name of each wrapped server

if the wrapped server is not on the same machine as the Xi wrapper.

 

Xi TextFiles

These files are examples of files that can be used to hold ItemIds of items that are to be included in a client application defined Xi List. The Client Base code can read text files to allow for manual configuration of the client Xi Lists.
